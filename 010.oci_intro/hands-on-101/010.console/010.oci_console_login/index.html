    • OCI 사용자 ID 관리 서비스

    이번 Hands-on Lab에서는 Oracle 계정을 등록한 다음에 Oracle Console에 로그인하는 방법을 소개합니다. 이번 Hands-on Lab에서 다음과 같은 정보로 Oracle Cloud 계정을 만들었다고 가정하고 진행하겠습니다.

    Account Name ocidemo001
    E-mail taewamne@gmail.com
    Password Welcome123456!
    Home Region Ashburn

    오라클 홈페이지에서 OCI Console 로그인

    Oracle 홈페이지에서 OCI Console은 https://oracle.comm 에서 로그인합니다. 오라클 홈페이지에서 로그인하는 방식은 Oracle IDCS를 사용하는 로그인 방식입니다. 오라클 홈페이지에서 다음과 같은 메뉴 패스로 오라클 클라우드 로그인 페이지로 이동합니다.

    View Accounts Sign in to Cloud

    Cloud Account Name 항목이 ocidemo001을 입력하고 Next 버튼을 클릭합니다.

    사용자 이름과 비밀번호에 항목에 각각 teawanme@gmail.com과 Welcome123456!을 입력하고 로그인합니다.

    로그인하면 OCI Classic 콘솔 페이지가 출력됩니다.

    OCI Classic 콘솔 페이지에서 다음과 같은 메뉴 패스로 OCI Console 로그인 페이지로 이동합니다.

    (콘솔 왼쪽 위 햄버그 버튼) Infrastructure

    OCI Console 로그인: Oracle IDCS 옵션 선택

    OCI Console 로그인 페이지에서는 Oracle IDCS와 OCI IAM 두 가지 옵션을 모두 제공합니다. Oracle IDCS 옵션을 선택할 경우 기존 Oracle Cloud의 SSO(싱글사인온)가 적용되고 OCI IAM을 선택할 경우 새로 ID/패스워드를 입력하고 인증합니다.

    현재는 Oracle IDCS 인증 방식을 선택합니다.

    앞에서 Oracle IDCS를 선택할 경우에 Oracle IDCS는 현재 사용자가 이미 로그인 상태라는 것은 SSO로 확인합니다. 따라서 추가 인증 정보를 요구하지 않고 바로 OCI Console로 이동합니다.

    현재 OCI Console에 로그인은 Oracle IDCS로 로그인했습니다. 따라서 OCI Console의 메뉴에는 Oracle Classic 기반 PaaS 서비스가 출력됩니다. 현재 이용가능한 OCI Classic PaaS 서비스는 다음 메뉴 패스로 확인할 수 있습니다.

    (콘솔 왼쪽 위 햄버그 버튼) 플랫폼 서비스

    OCI Console을 다음 메뉴 패스로 로그아웃합니다.

    사용자 프로파일 아이콘 사인 아웃

    사인 아웃 버튼을 클릭하면 다음과 같이 OCI Console 로그인 페이지로 이동합니다.

    OCI Console 로그인: OCI IAM 옵션 선택

    OCI Console 로그인 페이지에서 OCI IAM 인증을 위한 ID/패스워드를 입력하고 인증합니다.

    OCI IAM 인증을 하면 인증이 적용되는 범위는 OCI에 국한됩니다. 따라서 OCI Console 메뉴에서 OCI Classic 기반 PaaS 서비스 목록은 출력하지 않습니다.

    OCI Console을 다음 메뉴 패스로 로그아웃합니다.

    사용자 프로파일 아이콘 사인 아웃

    사인 아웃 버튼을 클릭하면 다음과 같이 OCI Console 로그인 페이지로 이동합니다.

    OCI 홈 리전 로그인 페이지

    앞에서 OCI Console에 접근하기 위해서 다음과 같은 경로를 거쳤습니다.

    oracle.com 홈페이지 oracle cloud 로그인 페이지 OCI Classic Console OCI Console 로그인 페이지 OCI Console

    이 방식은 약간 번거롭게 느껴집니다. oracle.com 홈페이지를 거치지 않고 직접 OCI Console 로그인 페이지에 접근할 수 있습니다. OCI Console 로그인 페이지는 OCI Account(Tenente)가 사용하는 Home Region에 따라서 달라집니다. OCI Console 로그인 페이지 패턴은 다음과 같습니다.

    https://console.<OCI_REGION_ID>.oraclecloud.com/
https://console.<OCI_REGION_ID>.oraclecloud.com/?tenant={Tenant_Name}

    OCI_REGION_ID는 다음 페이지에서 확인할 수 있습니다.

    https://console.ap-seoul-1.oraclecloud.com/?tenant=ocidemo001

    위 URL을 실행하면 다음 페이지가 출력됩니다.

    현재 Hands-on Lab에서 사용하는 ID는 ocidemo001 계정의 관리자 ID입니다. OCI Console 로그인 페이지에서 Oracle IDCS와 OCI IAM을 모두 선택하고 로그인할 수 있습니다.

    OCI Console 로그인 페이지에서 Oracle IDCS 로그인 방식을 사용할 경우, 현재 SSO 로그인한 상태라면 바로 OCI Console 페이지로 이동합니다. 반대로 Oracle IDCS가 SSO 로그인 상태가 아니라면 ID/Passowd를 입력하는 페이지로 이동합니다.

    OCI Console 로그인 페이지에서 OCI IAM 로그인 방식을 사용할 경우, 해당 페이지에서 ID/Password를 입력하고 OCI Console 페이지로 이동합니다.

    OCI Console 로그인 페이지에서 Oracle IDCS 혹은 OCI IAM으로 인증을 완료하면 다음과 같은 OCI Console 페이지가 출력됩니다.
